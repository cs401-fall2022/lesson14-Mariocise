TypeScript Tutorial

TypeScript is a language that adds static typing to JavaScript, designed to make code more predictable and therefore easier to debug.

Types in JavaScript

JavaScript is a dynamically typed language, meaning variables' types can change. This grants a lot of flexibility, but can lead to some problems. For example, function foo(x) { return (1 + x); } behaves differently depending on x's type. If x is a number, foo will return an number. If x is a string, foo will return a string. This flexibility can be useful, but also can become unpredictable, especially in a large code base.

Types in TypeScript

TypeScript is a strongly typed language, meaning variables' types don't change. The TypeScript equivalent of foo from earlier might look something like function foo(x: number) { return (1 + x); }. Since foo now only takes numbers for x, foo will always receive and return a number, which is much more predictable.

TypeScript can infer the type of a variable, or you can explicitly give a type. For example, let x = "one" and let x: string = "one" both do the same thing.

You can also define types. The following example defines Student as a type with an email (string) and studentid (number):

interface Student {
 email: string;
 studentid: number;
}

Structural Type System

Type checking in TypeScript considers the "shape" of values. If an object has the same "shape" as another object, it is considered to be of the same type. The object person1 in the example below is compatible with the type Student because it has the same "shape", and can be passed into functions requiring an object of type Student despite never being declared as a Student.

interface Student {
 email: string;
 studentid: number;
}
function getId(Student s) {
 return s.studentid;
}
const person1 = { email: "jimmy@website.com", studentid: 12345 };
getId(person1);
